Эта страница является шпаргалкой по командам Git

Вызов справки:

$ git краткая справка по git.

$ git help далее указывается имя команды. Вызывает справку по конкретной команде.


Глобальная настройка Git

используется только при первом запуске git на компьютере.


Создание репозитория:

$ git init инициирует пустой репозиторий в выбранной папке. В результате выполнения появится подкаталог .git в котором и будут храниться служебные настройки git'а и сам репозиторий./


Добавление файла в отслеживаемые:

$ git add имя существуещего файла в папке, где создан репозиторий (* добавляет все содержимое папки, либо файлы определенного расширения)

$ git add index.html
$ git add *
$ git add *.js

Проверка статуса файлов в репозитории:

$ git status позволяет посмотреть, какие файлы в папке отслеживаются, а какие - нет.


Коммит:

$ git commit -a -m после -m должен быть комментарий в кавычках.

$ git commit -a -m "Initial commit"

Удаление коммита:

$ git revert после указывается идентификатор коммита. Создает "зеркальный" коммит, который отменяет действие предыдущего.

$ git revert 4ad7e0

Исправление комментария последнего коммита:

$ git commit --amend -m после -m должен быть новый комментарий к последнему коммиту в кавычках.

$ git commit --amend -m "Roboto Font"

Просмотр истории коммитов:

$ git log показывает историю всех коммитов с их идентификаторами.

$ git commit -a -m "Initial commit"

Просмотр конкретного коммита:

$ git show вызывает конкретный коммит по его идентификатору. Можно использовать только первые 6 цифр идентификатора.

$ git show 4ad7e0

Взаимодействие с GitHub

Клонирование репозитория:

$ git clone после добавляется ссылка на репозиторий GitHub.

$ git clone https://github.com/netology-git/demo.git

Взаимодействие с удаленным репозиторием GitHub:

$ git remote позволяет управлять удалёнными репозиториями (добавлять, удалять, просматривать). Общепринято, что первый удалённый репозиторий называют origin. Напрмер, $ git remote -v позволяет просматривать удаленный репозиторий.

$ git remote -v
origin https://github.com/netology-git/demo.git (fetch)
origin https://github.com/netology-git/demo.git (push)

$ git remote add origin позволяет подключает удаленный репозиторий к локальному

.
$ git remote add origin https://github.com/netology-git/demo.git
$ git remote -v
origin https://github.com/netology-git/demo.git (fetch)
origin https://github.com/netology-git/demo.git (push)

$ git push отправляет изменения, сделанные в локальном репозитории на удаленный. Oigin и master необходио указывать только при первой отправке; в дальнейшем (и при наличии только одной ветки) можно указывать просто $ git push.

$ git push -u origin master
